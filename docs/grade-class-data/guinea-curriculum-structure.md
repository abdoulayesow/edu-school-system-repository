## üí° Implementation Notes for GSPN System

### Database Considerations

1. **Subject Codes**: Use standardized codes (e.g., `MATH`, `FRANCAIS`) for consistency
2. **Grade Levels**: Store both numeric (1-13) and French nomenclature (CP1, CP2, etc.)
3. **Specializations**: Track student series selection starting from Grade 12
4. **Optional Subjects**: Flag optional subjects in the curriculum table
5. **Exam Years**: Mark grades with national exams (6, 10, 13)

### Recommended Tables

```sql
-- Subject master table
CREATE TABLE subjects (
  id UUID PRIMARY KEY,
  code VARCHAR(20) UNIQUE NOT NULL,
  name_fr VARCHAR(100) NOT NULL,
  name_en VARCHAR(100) NOT NULL,
  is_optional BOOLEAN DEFAULT FALSE
);

-- Grade-subject mapping
CREATE TABLE grade_subjects (
  id UUID PRIMARY KEY,
  grade_level INTEGER NOT NULL, -- 1-13
  grade_code VARCHAR(10) NOT NULL, -- CP1, CP2, 7√®me, etc.
  subject_id UUID REFERENCES subjects(id),
  series VARCHAR(10), -- NULL for grades 1-11, A/B/C for 12-13
  coefficient INTEGER DEFAULT 1
);
```

### UI Considerations

- Display subjects grouped by grade level
- Show specialization options when students reach Grade 11
- Highlight exam years (CEPE, BEPC, BAC)
- Support both French and English subject names for i18n
